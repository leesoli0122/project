1. HTML 문법 (따옴표, 태그정의, 들여쓰기)

  • 모든 요소와 속성은 소문자로 작성한다.
  • 속성의 값은 큰따옴표("")로 묶는다.
  • 특수기호는 Entity code(ISO-8859-1)를 사용한다.
  • 종료태그를 반드시 선언한다. <img /> <br /> ...
  • 마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.

  • 다음의 경우에는 들여쓰기를 하지 않는다.

    • html의 자식 요소인 head, body
    • head의 자식 요소인 meta, link, script
    • body의 자식 첫 번째 요소 id="wrapper"

  • 마크업 유의사항

    • DTD 선언 앞에 주석 등의 내용이 없어야 한다.
    • <head>태그의 안쪽과 <body>태그의 안쪽 이외에 내용을 삽입하지 않는다.
    • a 태그에 삽입된 img 태그에는 이미지맵을 사용할 수 없다.
    • a 태그 안에는 a 태그가 들어갈 수 없다.
    • p 태그로는 레이아웃을 잡지 않는다. (p태그 안에는 블럭요소 삽입이 불가능하므로 운영시 문제가 생길 수 있음)
    • p 태그 안에는 p, div, ul 등 다른 블럭요소가 들어갈 수 없다. ( display:inline 속성을 적용해도 소용없음)
    • pre 태그 안에는 img, object, big, small, sub, sup 태그가 들어갈 수 없다.
    • button 태그 안에는 input, select, textarea, label, button, form, fieldset, iframe, isindex 태그가 들어갈 수 없다.
    • button 태그의 type 속성을 지정하지 않았을 때 IE6, IE7 의 경우 submit 기능으로 동작하므로 폼 값을 전송하는 용도로 사용하지 않을 경우 type="button"을 지정
    • form 태그 안에는 form 태그가 들어갈 수 없다.
    • label 태그 안에는 label 태그가 들어갈 수 없다.
    • id 값은 하나의 페이지에서 하나씩만 사용한다.

5. table 요소 사용 방법

  • table 요소의 속성 지정 순서는 cellspacing, summay 이며 border 값은 클래스를 지정하여 사용한다.
  • caption 요소는 표의 제목을 표현하며 표의 윗부분에 헤딩 요소로 표의 제목을 표시하면 생략가능하다.
  • 디자인에 머리글이 제공되어 있지 않더라도 반드시 머리글을 제공한다.
  • 행 제목과 열 제목이 있는 경우 이에 대한 scope 속성을 사용하여 행과 열의 범위를 적절히 구분하여 배치한다.

6. 요소의 기본속성 지정 순서

a 태그 요소

  • href 속성의 값은 #none으로 한다.
  • 앵커 기능이 있다면 해당 요소의 id값을 작성한다.
  • 새 창으로 페이지를 표시해야 할 경우 target 속성을 선택적으로 사용한다.
  • 링크의 목표가 명확하지 않은 경우 title 속성의 값으로 부가적인 설명을 제공한다.

img 요소 접근성

  • alt 속성은 반드시 제공되어야 하며 의미없는 이미지에 한하여 빈 값을 허용한다.
  • 유지보수의 편의성을 위하여 width, height 속성은 사용하지 않는다.
  • 대체 텍스트 사용으로 일부 브라우저에서 말 풍선이 나타는 것을 막을 필요가 있는 경우 title 속성의 빈 값을 제공한다.
  • 이미지 맵 사용을 지양하며 포지셔닝 방법을 사용한다.

CSS 작업규칙

1. 네이밍규칙

  • 이름은 영문 소문자로만 시작할 수 있다.
    • ex) btn-apply-on, box-news, box-reply-open 등
  • id는 camelcase 방식으로 사용한다.
  • 두개의 단어를 사용할 경우에는 이어지는 단어를 대문자로 표기하여 사용한다. (카멜표기법)
  • id는 화면에서 한번만 사용이 되므로, 고유기능을 쉽게 인지할 수 있도록 네이밍한다.
    • ex) id="btnSch" ( x ) id="btnGnbSearch" ( o )
  • class는 하이픈 방식으로 사용한다. (케밥표기법)
    • ex) class="btn-blackLine-on", class="link-view"
  • 유사한 용도인 경우 가장 뒤에 숫자를 붙여 사용할 수 있으며 첫번째 이름은 숫자를 생략할 수 있다.
    • ex) .list-benefit, .list-benefit-02, .list-benefit-03
  • 네이밍의 조합은 '형태-의미-순서-상태'을 기본 순서로 사용한다.
    • ex) .btn-confirm-02-on

2. css 작성규칙

  • 모든 속성은 영문 소문자로만 작성한다.
  • 폰트의 선언, 배경 url, filter, content에 큰 따옴표("")를 사용한다.
  • 마지막 선언된 속성에도 세미콜론(;)을 사용한다.
  • 선택자 간, 중괄호간 한칸의 공백을 사용한다, 중괄호 안쪽 좌우에는 공백은 사용한다.
  • 객체를 구분하기 위하여 코드 그룹 간 1줄의 빈 줄을 허용한다. 빈 줄은 1줄을 초과하지 않는다.
  • 중괄호( { , } ) 사이는 줄바꿈하지 않는다.
  • 두개 이상의 요소에 하나의 스타일을 지정하는 경우 쉼표(,) 뒤로 줄바꿈한다.
  • CSS 주석 처리는 시작 주석을 작성하며 종료 주석은 작성하지 않는다.
  • 미디어 타입은 공통 CSS 파일의 가장 아랫 부분에 선언한다. 또한 미디어 타입의 코드 작성시 1탭의 들여쓰기는 허용한다.

3. 속성의 선언 순서

속성은 위치 - 형태 - 흐름 - 크기 - 박스 - (배경 - 글자 - 기타) 순으로 지정한다.

전체 속성의 선언 순서는 시계 방향으로 명시한다. top, right, bottom, left


  • 1.위치 : position left right top bottom z-index
  • 2.형태 : display visibility
  • 3.흐름 : float clear
  • 4.크기 : width height overflow
  • 5.박스 : margin padding border
  • 6.배경 : background
  • 7.글자 : text font color
  • 8.기타 : cursor

  • px 단위 수치는 정수값 사용
